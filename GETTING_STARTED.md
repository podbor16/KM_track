## 📊 СТРУКТУРА ДАННЫХ

### Таблица `results` - основные результаты

Содержит один результат на участника:
- **Финишные времена**: `time_gun_finish`, `time_clear_finish`
- **Средний темп на финише**: `finish_pace_avg` (мм:сс мин/км)
- **Ранги**: абсолютный, по полу, по категории
- **Времена на КТ**: `time_clear_kt1`, `time_clear_kt2`, ...
- **Темпы на КТ**: `pace_avg_kt1`, `pace_avg_kt2`, ...
- **Статус**: Not started, Running, Finished, DNF, DSQ, Withdrawn

### Таблица `result_segments` - результаты по сегментам

Содержит результаты участников по отдельным сегментам маршрута:
- **Коды сегментов**: `start-kt1`, `kt1-kt2`, `kt2-finish`, и т.д.
- **Время на сегменте**: `sg_time_clear`
- **Темп на сегменте**: `sg_pace_avg`
- **Ранги на сегменте**: абсолютный, по полу, по категории

## 🔢 ФОРМУЛЫ РАСЧЁТОВ

### Темп (мм:сс мин/км)
```
Темп = (Время в секундах) / (Дистанция в км)

Пример: 7 км за 35:00 (2100 сек)
Темп = 2100 / 7 = 300 сек/км = 05:00 мин/км
```

### Ранги
- Берутся из API `race_data.json` (Copernico) в реальном времени
- Обновляются автоматически по мере финиша участников
- Считаются по чистому времени (`time_clear_finish`)

### Время на сегменте
```
Время сегмента = time_clear_конца - time_clear_начала

Пример: От КТ1 до КТ2
Время сегмента = time_clear_kt2 - time_clear_kt1
```

-- Топ-10 финишеров
SELECT rank_absolute, surname, name, time_clear_finish, finish_pace_avg
FROM results 
WHERE event_id = 1 AND race_status = 'Finished'
ORDER BY rank_absolute LIMIT 10;

-- Статистика по полам

FROM results
WHERE event_id = 1 AND race_status = 'Finished'

## 🎓 ЧТО ДЕЛАЕТ СКРИПТ

### На каждой итерации:

1. **Читает** `race_data.json` (обновляется fetcher'ом каждые 10 сек)
2. **Определяет** событие из поля `event` в JSON
3. **Получает** конфигурацию контрольных точек из таблицы `events`
4. **Для каждого участника:**
   - Получает/создает запись в таблице `results`
   - Преобразует времена из миллисекунд в TIME формат (HH:MM:SS)
   - Вычисляет средние темпы (дистанция / чистое время)
   - Получает ранги из `race_data.json`
   - Создает/обновляет сегменты в таблице `result_segments`
5. **Фиксирует** все изменения в БД
6. **Выводит** статистику (сколько успешно, сколько ошибок)
7. **Ждёт** N секунд перед следующей итерацией (если --continuous)

### Кэширование для оптимизации:
- ✅ События (не перезапрашиваются если есть в памяти)
- ✅ Клиенты (не ищутся повторно)
- ✅ Результаты (не переделываются, если уже есть)
