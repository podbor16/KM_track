## Контекст

Страница `/24h` уже имеет три вкладки: Расстояние, Отрезки, График. Добавляем четвёртую — «Круги» — для анализа произвольного диапазона кругов по каждому участнику. Данные в БД уже есть (`laps.lap_ms`, `laps.cumulative_ms`), эндпоинт `/api/tri/laps` уже существует.

---

## Что строим

### 1. Новая вкладка «Круги» (4-я на странице)

Таблица участников с агрегатом за выбранный диапазон кругов. Раскрываемые строки с детализацией по каждому кругу.

### 2. Подвкладка «По кругам» в «Графике» (4-я в График)

Линейный график lap_ms по номеру круга для выбранных участников. Реализуется отдельно, в рамках этого спека не описывается подробно.

## Компоненты вкладки «Круги»

### Фильтр категории

Те же чипы Все / Мужчины / Женщины / Эстафета что и на других вкладках. Состояние фильтра **синхронизировано** между вкладками через общую переменную `currentCategory`.

### Слайдер диапазона

Двухручечный слайдер (аналог слайдера времени на «Графике»). Диапазон от круга 1 до текущего максимального круга среди всех участников.

- Подпись над слайдером: «Круги X — Y»
- Под слайдером: «Показано N кругов · M км · период Ч:ММ»
  - N = количество кругов в диапазоне
  - M = N × 4.040 (длина круга)
  - период = cumulative_ms круга Y − cumulative_ms круга X у участника с наибольшим числом кругов в диапазоне
- При SSE-обновлении максимум слайдера обновляется, позиция ручек **не сбрасывается**

### Таблица (свёрнутый вид)

Сортировка по умолчанию: **лучший круг в диапазоне по возрастанию** (быстрее → выше).

Колонки:

| # | Участник | Кругов | Лучший ↑ | Худший | Средний | Разброс | ▶ |
|---|----------|--------|----------|--------|---------|---------|---|

- **#** — порядковый номер в этой таблице (не место в общем зачёте)
- **Участник** — фамилия имя + тег М/Ж, для DNF — плашка `DNF`, строка приглушена (opacity 0.55), в конце таблицы
- **Кругов** — количество кругов участника, попавших в диапазон [X, Y]
- **Лучший** — min(lap_ms) за диапазон, зелёный цвет
- **Худший** — max(lap_ms) за диапазон, красный цвет
- **Средний** — avg(lap_ms) за диапазон
- **Разброс** — max−min, формат ±М:СС
- **▶** — иконка раскрытия, поворачивается на 90° при открытии

### Раскрытая строка (детализация)

Вложенная таблица со всеми кругами участника в диапазоне:

| Круг | Время | Скорость | ±от среднего | Нарастающее |
|------|-------|----------|--------------|-------------|

- **Круг** — lap_number
- **Время** — lap_ms в формате ММ:СС. Лучший круг отмечен ★ и зелёным
- **Скорость** — 4.040 / (lap_ms / 3600000) км/ч, 1 знак
- **±от среднего** — lap_ms − avg_lap_ms, зелёный если отрицательный (быстрее), красный если положительный
- **Нарастающее** — cumulative_ms в формате Ч:ММ:СС

## Данные и эндпоинт

### Источник данных

`GET /api/tri/laps` — уже существует, возвращает:

```json
{
  "laps": [
    {
      "participant_id": 1,
      "lap_number": 10,
      "lap_ms": 862000,
      "cumulative_ms": 8640000,
      "surname": "Иванов",
      "name": "Иван",
      "start_number": 5
    }
  ]
}
```

**Решение:** фронт джойнит laps со standings по `participant_id` — standings уже содержат gender, category, status (DNF). Отдельного изменения бэкенда не требуется.

### SSE-обновления

Вкладка «Круги» перезапрашивает `/api/tri/laps` при каждом SSE-событии обновления данных (тот же обработчик что у standings). Агрегаты пересчитываются, позиция слайдера не сбрасывается, раскрытые строки восстанавливаются через `openLapsRows`.

## Фронтенд — структура кода

Всё в `tri_results.html` (как остальные вкладки).

### Новые функции

```
initLapsTab()          — первичная загрузка данных, инициализация слайдера
renderLapsTable()      — отрисовка таблицы по текущему диапазону и фильтру
aggregateForRange(laps, pidList, from, to)  — агрегат по участнику за диапазон
toggleLapsRow(pid)     — раскрытие/закрытие строки
onSliderChange()       — обработчик слайдера, вызывает renderLapsTable()
```

### Слайдер

Реализовать на нативном `<input type="range">` с двумя overlapping инпутами (паттерн multi-thumb range slider без библиотек) — аналогично существующему слайдеру на вкладке «График».

### Состояние

```js
let lapsData = [];        // сырые данные из /api/tri/laps
let lapsRangeFrom = 1;    // текущая левая граница
let lapsRangeTo = null;   // текущая правая граница (null = максимум)
let openLapsRows = new Set(); // раскрытые participant_id
```
