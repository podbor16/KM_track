# Вкладка «Подготовка стартовых» — DataLens Implementation Plan (Wizard)

> **Подход:** обычные Wizard-чарты через датасет с вычисляемыми полями. Никакого SQL в редакторе чартов.

**Goal:** Создать вкладку «Подготовка стартовых» в существующем дашборде DataLens с 5 проверками регистраций (10 виджетов: 5 индикаторов + 5 таблиц).

**Architecture:** Один датасет `ds_leads_checks` (таблица `leads`) с вычисляемыми полями-флагами (0/1) и текстовыми полями-описаниями нарушений. Индикаторы показывают `SUM(flag)`. Таблицы фильтруются по `flag = 1`. Два Selector-виджета фильтруют все чарты через механизм связей дашборда.

**Tech Stack:** Yandex DataLens — Dataset Calculated Fields, Wizard Charts (Indicator, Table), Selectors, Dashboard Tabs.

**Спек:** `docs/superpowers/specs/2026-05-15-start-list-prep-design.md`

## Задача 1: Датасет и вычисляемые поля

*Все проверки реализуются как вычисляемые поля в датасете. Чарты только отображают — никакой логики в чартах.*

- [ ] **Шаг 1: Открыть датасет `ds_leads_checks`**

DataLens → Коллекции → Красмарафон → `ds_leads_checks` → Редактировать.  
Перейти на вкладку **«Поля»** → кнопка **«Добавить поле»**.

- [ ] **Шаг 2: Добавить поле `age`** (возраст участника)

Название: `age`  
Тип: Целое число  
Формула:
```
[event_year] - YEAR([birthday])
```

- [ ] **Шаг 3: Добавить поле `full_name`**

Название: `full_name`  
Тип: Строка  
Формула:
```
CONCAT([surname], " ", [name])
```

- [ ] **Шаг 4: Добавить поле `birthday_year`**

Название: `birthday_year`  
Тип: Целое число  
Формула:
```
YEAR([birthday])
```

- [ ] **Шаг 5: Добавить поле `льготная_нарушение`** (флаг 1/0)

Название: `льготная_нарушение`  
Тип: Целое число  
Формула:
```
IF(
  CONTAINS([products], "Дети") AND [age] >= 18, 1,
  IF(
    CONTAINS([products], "Пенсионер") AND [sex] = "М" AND [age] < 60, 1,
    IF(
      CONTAINS([products], "Пенсионер") AND [sex] = "Ж" AND [age] < 55, 1,
      IF(
        CONTAINS([products], "18-22") AND ([age] < 18 OR [age] > 22), 1,
        0
      )
    )
  )
)
```

- [ ] **Шаг 6: Добавить поле `льготная_несоответствие`** (текст для таблицы)

Название: `льготная_несоответствие`  
Тип: Строка  
Формула:
```
IF(
  CONTAINS([products], "Дети") AND [age] >= 18, "Возраст >= 18",
  IF(
    CONTAINS([products], "Пенсионер") AND [sex] = "М" AND [age] < 60, "М < 60 лет",
    IF(
      CONTAINS([products], "Пенсионер") AND [sex] = "Ж" AND [age] < 55, "Ж < 55 лет",
      IF(
        CONTAINS([products], "18-22") AND ([age] < 18 OR [age] > 22), "Возраст не 18-22",
        NULL
      )
    )
  )
)
```

- [ ] **Шаг 7: Добавить поле `birthday_error`** (флаг 1/0)

Название: `birthday_error`  
Тип: Целое число  
Формула:
```
IF([birthday_year] < 1900 OR [birthday_year] > [event_year], 1, 0)
```

- [ ] **Шаг 8: Добавить поле `birthday_проблема`** (текст для таблицы)

Название: `birthday_проблема`  
Тип: Строка  
Формула:
```
IF(
  [birthday_year] < 1900, "Год < 1900",
  IF([birthday_year] > [event_year], "Год > года события", NULL)
)
```

- [ ] **Шаг 9: Добавить поле `min_age`** (минимальный допустимый возраст по дистанции)

Название: `min_age`  
Тип: Целое число  
Формула:
```
IF(
  CONTAINS([event_distance], "21"), 18,
  IF(
    CONTAINS([event_distance], "Северная ходьба") AND [event_name] = "Жара", 7,
    IF(
      CONTAINS([event_distance], "Северная ходьба"), 12,
      IF(
        CONTAINS([event_distance], "2 км"), 7,
        IF(
          CONTAINS([event_distance], "5 км") OR CONTAINS([event_distance], "7 км") OR CONTAINS([event_distance], "10 км"), 12,
          0
        )
      )
    )
  )
)
```

- [ ] **Шаг 10: Добавить поле `min_age_нарушение`** (флаг 1/0)

Название: `min_age_нарушение`  
Тип: Целое число  
Формула:
```
IF([age] < [min_age] AND [min_age] > 0, 1, 0)
```

- [ ] **Шаг 11: Добавить поле `min_age_текст`** (текст нарушения для таблицы)

Название: `min_age_текст`  
Тип: Строка  
Формула:
```
IF(
  [age] < [min_age] AND [min_age] > 0,
  CONCAT("- ", STR([min_age] - [age]), " лет"),
  NULL
)
```

- [ ] **Шаг 12: Сохранить датасет**

Нажать «Сохранить». Убедиться что все поля без ошибок (красных восклицательных знаков).

## Примечания по отладке

**Поле `age` возвращает неверные значения:**  
→ Проверить тип поля `birthday` в датасете — должен быть «Дата», не «Строка». Если строка — поменять тип в настройках поля.

**Поле с формулой показывает ошибку «неизвестное поле»:**  
→ Убедиться что поля `age`, `birthday_year` созданы раньше полей которые на них ссылаются.

**Индикатор показывает дробное число:**  
→ Поменять агрегацию с «Сумма» на «Количество» или убедиться что тип поля-флага — целое число.

**Фильтр по `is_duplicate = 1` не работает:**  
→ Проверить тип поля `is_duplicate` в датасете. Если «Логический (boolean)» — фильтровать по «Истина», не по `1`.

**Таблица показывает агрегированные строки вместо отдельных записей:**  
→ Перенести поле `id` в секцию «Столбцы» первым — это заставит таблицу разворачивать по каждой заявке отдельно.
