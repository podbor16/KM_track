# 📋 Страница профиля спортсмена - Документация

## Описание функциональности

На странице истории (`/history`) была реализована新功能: при вводе имени спортсмена в поле поиска и выборе его из списка результатов, пользователь переходит на персональную страницу профиля спортсмена (`/athlete-profile`).

## Реализованные компоненты

### 1. **Функция в БД** (`src/analytics/db_connection.py`)
```python
def get_athlete_results(surname: str, name: str) -> tuple:
```
- Получает информацию о спортсмене из таблицы results
- Возвращает кортеж: (информация о спортсмене, список всех его результатов забегов)
- Поиск по внешнему ключу (фамилия и имя)

### 2. **API Эндпоинт** (`src/tracker/router.py`)
```
GET /api/athlete/{surname}/{name}
```
- Параметры: фамилия и имя спортсмена (URL encoded)
- Возвращает JSON с информацией о спортсмене и всеми его результатами
- Поддерживает форматирование datetime объектов в ISO формат

Пример запроса:
```
GET /api/athlete/Иванов/Иван
```

Пример ответа:
```json
{
  "athlete": {
    "surname": "Иванов",
    "name": "Иван",
    "birthdate": "1995-05-15",
    "gender": "male",
    "city": "Москва"
  },
  "results": [
    {
      "surname": "Иванов",
      "name": "Иван",
      "event": "5 km",
      "status": "finished",
      "times.official_:::finish:::": 1800000,
      "category": "Мужчины",
      "rankings_:::full-1:::": 42,
      "gunTime": "2025-12-07T04:00:00"
    }
  ],
  "total_races": 5
}
```

### 3. **HTML Страница** (`legacy/templates/athlete-profile.html`)
- Адрес: `/athlete-profile?surname={surname}&name={name}`
- Отображает:
  - Информацию о спортсмене (ФИ, год рождения, пол, город)
  - Статистику (всего забегов, финишировал, пройдено км)
  - Историю забегов в двух режимах просмотра:
    - **Сетка** (Grid view) - красивые карточки каждого забега
    - **Таблица** (Table view) - классический табличный формат

### 4. **JavaScript на странице истории** (`legacy/templates/history.html`)
- Функция `selectAthlete()` обновлена для перехода на страницу профиля
- При клике на спортсмена в результатах поиска выполняется редирект на `/athlete-profile`

## Маршруты в приложении

### HTML страницы
- `/history` - Страница истории с поиском спортсменов
- `/athlete-profile` - Страница профиля спортсмена

### API эндпоинты
- `GET /api/search-athletes?q=...` - Поиск спортсменов
- `GET /api/athlete/{surname}/{name}` - Получение информации и результатов спортсмена

## Примеры использования

### Шаг 1: Открыть страницу истории
```
http://localhost:8000/history
```

### Шаг 2: Ввести текст для поиска
- В поле "Введите фамилию или имя спортсмена..." постепенно появляются результаты
- Нажать минимум 2 символа для начала поиска

### Шаг 3: Выбрать спортсмена
- Кликнуть на нужного спортсмена в списке результатов
- Автоматический редирект на страницу профиля

### Шаг 4: Просмотр результатов
На странице профиля доступны:
- Личная статистика спортсмена
- История всех его забегов
- Детали каждого забега (время, темп, место)
- Переключение между видами отображения (сетка/таблица)

## Техническая информация

### Использованные технологии
- **Backend**: FastAPI, MySQL, Python
- **Frontend**: HTML5, JavaScript, CSS3
- **Шаблонизация**: Jinja2

### Структура данных результата
Из таблицы `results` используются следующие поля:
- `surname` - Фамилия
- `name` - Имя
- `birthdate` - Дата рождения
- `gender` - Пол
- `city` - Город
- `event` - Дистанция забега (например "5 km")
- `status` - Статус (finished, running, notstarted, disqualified)
- `times.official_:::finish:::` - Время финиша (в миллисекундах)
- `category` - Возрастная категория
- `rankings_:::full-1:::` - Место финиша
- `gunTime` - Время старта забега
- Другие служебные поля

### Фильтрация и сортировка
На странице профиля результаты сортируются по дате в обратном порядке (новые сверху).

## Возможные расширения

1. **Добавить фильтры** по дате, дистанции, статусу
2. **Добавить графики** прогресса спортсмена
3. **Сравнение** с другими спортсменами
4. **Экспорт** результатов в PDF/Excel
5. **Статистика** по улучшению времени, темпа

## Известные особенности

- Поиск работает по полному совпадению фамилии и имени
- Если спортсмен не найден, отображается ошибка 404
- Очень большое количество результатов может замедлить загрузку страницы
- Необходимо правильное подключение к базе данных (переменные окружения DB_*)
